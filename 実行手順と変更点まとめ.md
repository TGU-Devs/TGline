## 実行手順

### 手順1: プロジェクトディレクトリに移動

PowerShellまたはコマンドプロンプトを開き、プロジェクトのルートディレクトリに移動します。

```powershell
# 例: プロジェクトがc:\書類\Bulletin-board-にある場合
cd c:\書類\Bulletin-board-
```

**⚠️ 注意**: Windowsで日本語パスを含む場合、PowerShellが正しく動作しない可能性があります。
その場合は、後述の「変更点3」を参照してください。

---

### 手順2: backend/.envファイルを作成

`backend`ディレクトリ内に`.env`ファイルを作成します。

#### 方法A: PowerShellで自動作成

```powershell
@"
# Database configuration
DATABASE_HOST=db
DATABASE_USER=postgres
DATABASE_PASSWORD=postgres
DATABASE_NAME=backend_development

# Rails configuration
RAILS_MAX_THREADS=5

# Frontend API URL
NEXT_PUBLIC_API_URL=http://localhost:3001
"@ | Out-File -FilePath backend\.env -Encoding utf8
```

#### 方法B: 手動作成

1. `backend`フォルダを開く
2. 新しいテキストファイルを作成し、ファイル名を`.env`に変更
3. 以下の内容をコピー&ペースト:

```
# Database configuration
DATABASE_HOST=db
DATABASE_USER=postgres
DATABASE_PASSWORD=postgres
DATABASE_NAME=backend_development

# Rails configuration
RAILS_MAX_THREADS=5

# Frontend API URL
NEXT_PUBLIC_API_URL=http://localhost:3001
```

4. 保存（UTF-8エンコーディングで保存すること）

---

### 手順3: docker-compose.ymlの修正

プロジェクトルートの`docker-compose.yml`ファイルを修正します。

#### 修正箇所1: dbサービスの環境変数（3-12行目）

**変更前:**
```yaml
  db:
    image: postgres:15
    env_file:
      - ./backend/.env
    environment:
      POSTGRES_HOST_AUTH_METHOD: md5
    ports:
      - "5432:5432"
    volumes:
      - db_data:/var/lib/postgresql/data
```

**変更後:**
```yaml
  db:
    image: postgres:15
    env_file:
      - ./backend/.env
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
      POSTGRES_DB: backend_development
      POSTGRES_HOST_AUTH_METHOD: md5
    ports:
      - "5432:5432"
    volumes:
      - db_data:/var/lib/postgresql/data
```

#### 修正箇所2: frontendサービスのvolumes（43行目）

**変更前:**
```yaml
    volumes:
      - ./frontend:/app
       - /app/node_modules  # ← インデントが間違っている
```

**変更後:**
```yaml
    volumes:
      - ./frontend:/app
      - /app/node_modules  # ← インデントを修正
```

#### 修正箇所3: frontendサービスのenv_file（46-47行目）

**変更前:**
```yaml
    env_file:
      - backend/.env  # ← パスが間違っている
```

**変更後:**
```yaml
    env_file:
      - ./backend/.env  # ← ./を追加
```

---

### 手順4: Dockerコンテナをビルド・起動

プロジェクトルートディレクトリで以下のコマンドを実行します。

```powershell
docker compose up --build
```

**初回起動時の目安時間**: 5～10分程度（イメージのダウンロードとビルドに時間がかかります）

---

### 手順5: 起動確認

ターミナルに以下のようなメッセージが表示されれば起動成功です:

```
bulletin-board-db-1       | database system is ready to accept connections
bulletin-board-backend-1  | * Listening on http://0.0.0.0:3000
bulletin-board-frontend-1 | ✓ Ready in 2.5s
bulletin-board-frontend-1 | ○ Local:   http://localhost:3000
```

---

### 手順6: ブラウザでアクセス

以下のURLにアクセスして動作確認します:

- **フロントエンド**: http://localhost:3000
- **バックエンドAPI**: http://localhost:3001/api/time

フロントエンドに現在時刻が表示されれば成功です！

## Readme.mdとの差異・変更点

### 🔴 変更点1: `.env`ファイルが存在しない

**問題点:**
- Readme.mdでは`.env`ファイルの作成手順が記載されていますが、実際のプロジェクトには`.env`ファイルが存在していません

**対応:**
- 上記「手順2」で`.env`ファイルを必ず作成する必要があります
- `.env`ファイルは`.gitignore`に含まれているため、Gitリポジトリには含まれません
- **各開発者が個別に作成する必要があります**

---

### 🔴 変更点2: `docker-compose.yml`のデータベース環境変数不足（重要）

**問題点:**
- `db`サービスに`POSTGRES_PASSWORD`などの必須環境変数が設定されていない
- このままだとPostgreSQLコンテナが起動時にエラーで停止します

**修正内容:**
```yaml
# 修正前（エラー）
  db:
    image: postgres:15
    env_file:
      - ./backend/.env
    environment:
      POSTGRES_HOST_AUTH_METHOD: md5

# 修正後（正常）
  db:
    image: postgres:15
    env_file:
      - ./backend/.env
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
      POSTGRES_DB: backend_development
      POSTGRES_HOST_AUTH_METHOD: md5
```

**影響:**
- この修正をしないと、データベースコンテナが以下のエラーで起動失敗します:
  ```
  Error: Database is uninitialized and superuser password is not specified.
  You must specify POSTGRES_PASSWORD to a non-empty value for the superuser.
  ```

---

### 🔴 変更点3: `docker-compose.yml`のインデントエラー

**問題点:**
- 43行目の`- /app/node_modules`のインデント（字下げ）が1つ多い
- このままだとYAMLの構文エラーでDocker Composeが起動しません

**修正内容:**
```yaml
# 修正前（エラー）
    volumes:
      - ./frontend:/app
       - /app/node_modules  # ← スペースが1つ多い

# 修正後（正常）
    volumes:
      - ./frontend:/app
      - /app/node_modules  # ← 正しいインデント
```

**影響:**
- この修正をしないと、`docker compose up`実行時にYAML構文エラーが発生します

### 🔴 変更点4: `docker-compose.yml`のパス指定エラー

**問題点:**
- frontendサービスの`env_file`が`backend/.env`となっており、相対パスの指定が不完全
- Docker Composeは`./backend/.env`の形式を推奨

**修正内容:**
```yaml
# 修正前
env_file:
  - backend/.env

# 修正後
env_file:
  - ./backend/.env
```

**影響:**
- 環境によっては`.env`ファイルが読み込めない可能性があります

### 🟡 注意点1: Windowsの日本語パス問題（Readme.mdに記載なし）

**問題点:**
- プロジェクトパスが`c:\書類\Bulletin-board-`のように日本語を含む場合、PowerShellでcd実行時にエラーが発生する可能性があります
- 実際に以下のエラーが発生しています:
  ```
  Set-Location : パス 'C:\書類\Bulletin-board-' が存在しないため検出できません。
  ```

**対処方法（以下のいずれか）:**

#### 対処法A: コマンドプロンプトを使用
PowerShellではなく、コマンドプロンプト（cmd.exe）を使用します。

```cmd
cd c:\書類\Bulletin-board-
docker compose up --build
```

#### 対処法B: プロジェクトを英語パスに移動
プロジェクトを英語のみのパスに移動します。

```powershell
# 例: C:\Projectsに移動
cd C:\
mkdir Projects
xcopy /E /I "c:\書類\Bulletin-board-" "C:\Projects\Bulletin-board"
cd C:\Projects\Bulletin-board
```

#### 対処法C: Git Bashを使用
Git for Windowsに含まれる「Git Bash」を使用します。

```bash
cd /c/書類/Bulletin-board-
docker compose up --build
```

**推奨:** 対処法Bの英語パスへの移動が最も確実です。

### 🟡 注意点2: VSCode依存の機能について（Readme.mdに記載あり）

Readme.mdでは「VSCode推奨」と記載されていますが、**実際にはVSCode依存の機能はありません**。

- Docker ComposeとDockerfileのみで動作します
- 任意のエディタ（メモ帳、Notepad++、サクラエディタなど）で開発可能
- ただし、以下の理由でVSCodeの使用を推奨します:
  - Dockerコンテナ内のファイル編集が容易（Dev Containers拡張機能）
  - ターミナル統合により、コマンド実行がスムーズ
  - Git統合により、バージョン管理が容易

**結論:** VSCodeは推奨されていますが、必須ではありません。

### エラー1: ポートが既に使用されている

```
Error: bind: address already in use
```

**解決方法:**
```powershell
# 既存のコンテナを停止
docker compose down

# ポート使用状況を確認
netstat -ano | findstr :3000
netstat -ano | findstr :3001
```

### エラー2: YAMLの構文エラー

```
yaml: line X: mapping values are not allowed in this context
```

**解決方法:**
- 上記「変更点2」のインデント修正を確認してください

### エラー3: `.env`ファイルが見つからない

```
env file backend/.env not found
```

**解決方法:**
- 上記「手順2」で`.env`ファイルを作成してください
