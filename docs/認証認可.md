# TGU掲示板 認証・認可設計

## 概要

- 認証方式：Devise + JWT（HttpOnly Cookie）
- 認可方式：ロールベース + リソースオーナーチェック（自前 concern）
- OAuth：Google ログイン対応

---

## 認証（Authentication）

### 認証フロー

```
1. ユーザーがログイン/サインアップ/Google ログイン
2. サーバーが JWT を生成し、HttpOnly Cookie にセット
3. 以降のリクエストで Cookie が自動送信される
4. サーバーが Cookie から JWT を取得・検証し、ユーザーを特定
```

### JWT 仕様

| 項目 | 値 |
|------|-----|
| アルゴリズム | HS256 |
| 秘密鍵 | `Rails.application.secret_key_base` |
| 有効期限 | 7日 |
| ペイロード | `{ user_id, exp }` |
| 保存場所 | Cookie（`jwt_token`） |

### Cookie 設定

| 属性 | 値 | 目的 |
|------|-----|------|
| `httponly` | `true` | XSS 対策（JS からアクセス不可） |
| `same_site` | `:lax` | CSRF 対策 |
| `secure` | `true`（本番のみ） | HTTPS 通信のみで送信 |
| 有効期限 | 7日 | JWT と同期 |

### 認証の実装箇所

- `app/controllers/concerns/authenticable.rb` — 認証ロジック
- `lib/jwt_service.rb` — JWT のエンコード/デコード
- `ApplicationController` に `Authenticable` が include され、全エンドポイントに `before_action :authenticate_user!` が適用

### 認証不要なエンドポイント

以下は `skip_before_action :authenticate_user!` で認証をスキップ：

| エンドポイント | コントローラー |
|------------|------------|
| `POST /api/users/sign_up` | RegistrationsController |
| `POST /api/users/sign_in` | SessionsController |
| `POST /api/users/google_sign_in` | GoogleSessionsController |
| `DELETE /api/users/sign_out` | SessionsController |
| `GET /api/users` | UsersController |
| `GET /api/users/:id` | UsersController |

### 認証エラーレスポンス

| ケース | ステータス | レスポンス |
|-------|----------|----------|
| トークンなし/無効 | 401 | `{ "error": "unauthorized" }` |
| トークン期限切れ | 401 | `{ "error": "token_expired" }` |

## 認可（Authorization）

### ロール定義

User モデルの `role` カラム（enum）：

| ロール | 説明 |
|-------|------|
| `user` | 一般ユーザー（デフォルト） |
| `admin` | 管理者 |

### 認可メソッド

`app/controllers/concerns/authorizable.rb` に定義：

| メソッド | 説明 | 失敗時 |
|---------|------|--------|
| `authorize_admin!` | 管理者のみ許可 | 403 Forbidden |
| `authorize_owner!` | リソース所有者のみ許可 | 403 Forbidden |
| `authorize_owner_or_admin!` | 所有者 or 管理者を許可 | 403 Forbidden |
| `authorize_roles!(*roles)` | 指定ロールのいずれかを許可 | 403 Forbidden |

ヘルパーメソッド：

| メソッド | 説明 |
|---------|------|
| `admin?` | 現在のユーザーが管理者か |
| `owner?(resource)` | 現在のユーザーがリソース所有者か |
| `owner_or_admin?(resource)` | 所有者 or 管理者か |

### リソースごとの認可ルール

| リソース | 操作 | 認可 |
|---------|------|------|
| 投稿 | 更新 | オーナーのみ |
| 投稿 | 削除 | オーナー or 管理者 |
| コメント | 削除 | オーナー or 管理者 |
| 管理画面 (`/api/admin/*`) | 全操作 | 管理者のみ |

### 認可エラーレスポンス

```json
{
  "error": "forbidden"
}
```

ステータス：403 Forbidden

## Google OAuth

### フロー

```
1. フロントエンドで Google Sign-In → ID トークン取得
2. ID トークンを POST /api/users/google_sign_in に送信
3. サーバーが google-id-token gem で検証
4. ユーザーを (provider, uid) で検索
   → 存在すればログイン（論理削除済みなら復元）
   → 存在しなければ新規作成
5. JWT を発行し Cookie にセット
```

### 実装箇所

- `lib/google_auth_service.rb` — ID トークン検証
- `app/controllers/api/users/google_sessions_controller.rb` — OAuth ログイン処理

### メール衝突の防止

同じメールアドレスがすでにメール/パスワード認証で登録されている場合、Google OAuth でのアカウント作成を拒否する（`EmailAlreadyTaken` エラー）。

### OAuth ユーザーの制約

- パスワードのバリデーションをスキップ（`password_required?` で判定）
- パスワード変更を拒否（PasswordsController で制御）
- アカウント削除時のパスワード確認をスキップ

### 環境変数

| 変数名 | 用途 |
|--------|------|
| `GOOGLE_CLIENT_ID` | Google OAuth クライアント ID |

## セキュリティまとめ

| 対策 | 実装 |
|------|------|
| XSS | HttpOnly Cookie |
| CSRF | SameSite=Lax |
| 通信暗号化 | Secure Cookie（本番） |
| パスワード保護 | bcrypt（12 stretches） |
| メールハイジャック | OAuth 時のメール重複チェック |
| 論理削除 | `deleted_at` による soft delete |
