**理由:** 
- 環境変数ファイルが存在しなかった
- `.gitignore`に含まれるため、各開発者が作成する必要がある

---

### 2. docker-compose.ymlの修正（3箇所） ✅

#### 修正1: dbサービスの環境変数追加（最重要）

```yaml
# 修正前
  db:
    image: postgres:15
    env_file:
      - ./backend/.env
    environment:
      POSTGRES_HOST_AUTH_METHOD: md5

# 修正後
  db:
    image: postgres:15
    env_file:
      - ./backend/.env
    environment:
      POSTGRES_USER: postgres          # ← 追加
      POSTGRES_PASSWORD: postgres      # ← 追加
      POSTGRES_DB: backend_development # ← 追加
      POSTGRES_HOST_AUTH_METHOD: md5
```

**エラー内容:**
```
Error: Database is uninitialized and superuser password is not specified.
You must specify POSTGRES_PASSWORD to a non-empty value for the superuser.
```

**理由:** PostgreSQLコンテナは`POSTGRES_PASSWORD`が必須

---

#### 修正2: frontendサービスのインデント修正

```yaml
# 修正前（エラー）
    volumes:
      - ./frontend:/app
       - /app/node_modules  # ← インデントが1つ多い

# 修正後（正常）
    volumes:
      - ./frontend:/app
      - /app/node_modules  # ← 修正
```

**理由:** YAMLの構文エラー

---

#### 修正3: frontendサービスのパス修正

```yaml
# 修正前
    env_file:
      - backend/.env

# 修正後
    env_file:
      - ./backend/.env  # ← ./を追加
```

**理由:** 相対パスの明示的な指定
